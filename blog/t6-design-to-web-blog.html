Re-designing for Responsiveness (Technical Blog)

What is a responsive site and why is it important?

Responsive sites are sites that are designed to create an optimal experience for the user, no-matter what type of device they are using. This is important as screen resolution varies considerably across devices. A responsive site will change to accommodate the change in available space. A responsive site should be easy to read and navigate with a minimum amount of scrolling anf re-sizing. Generally the content would be the same, but in some cases a smaller screen may have simpler content while a larger screen might show a more fully featured version.

What is mobile first, and why is it important?

'Mobile first' is a design philosophy where the designer will start by designing the simpler mobile version of the site, to ensure that this is slick and efficient, and then move on to the design for larger screens, as opposed to first designing the full-sized version of the site and the progressively "degrading" the site until it works well on a mobile device with a small screen. This is believed to be a better way of designing as the designer is forced to consider what is really important to include in a mobile version, rather than just including everything as there is plenty of space. This should produce a better hierachy of the content. Of course, it should be possible to start from either end and still do each properly and with the correct attention to the usability on a variety of screen sizes. Maybe the real problem here is that due to time and budget constraints, there will always be a compromise

What are frameworks? What are their pros and cons?

A framework is a collection of CSS classes that make a page layout design easier to implement. The framework supplies many of the building blocks for styles and helper-classes to easliy create the design.

Pros

  • much quicker to develop as a lot of the work has already been done
  • the code should be well-written and throughly tested
  • cross-browser issues should be handled by the framework, so your site should look good on all browsers
  • it will encourage consistency accross your site
  • there are plenty of frameworks to choose from, so there should be one suitable for most uses

Cons

  • a framework can be complex to use
  • using a framework could mean loading a lot of CSS that is never used
  • less flexible that hand-written CSS

What is a wireframe is and why would we use one?

A wireframe is a graphical representation of a page layout or the GUI, and the way that a user or visitor to a site will interact with it. The placement of content is shown, and how the user will navigate around the site.

What aspects of my wireframes were difficult to implement, and why?

There were lots of little things that I found frustrating as they all needed Javascript to implement. As this was not part of this weeks assignment, I made a number of changes to my design (3 to 4 revisions) to simplify it and still make it easy to navigate.

What accessibility features have you included in your site?

  • Aria roles have been added where possible.
  • Alternative text has been added to images.
  • Colour contrast on the pages has been checked using the Chrome Developer Accessibility Audit Tool

What usability features have you included in your site?

  • The site is clearly named on the home page.
  • A tagline has been used to describe the purpose of the site.
  • Page names match the text from the link.
  • There is a HOME link on every subsequent page, and it is always in the same place.
  • Global navigation is the same on every page
  • The content has a clear hierarchy.
