About Responsiveness

Grid based design

I'm finding the lack of responsiveness on so many webpages so irritating! The more I use my phone to access the web, the more sites I find that are so difficult to use. While I realise that responsive sites can have problems with download speeds, it seems like a really important improvement to make to an original site. A responsive webpage responds to its environment, changing its layout based on the width of the browser window as it is rendered. There is no need to specify a particular layout, as the page re-adjusts the size of each section on the fly. Awkward scrolling and re-sizing can be avoided. So at the moment, I'm all in favour of grid-based designs. Grid-based design is a way of dividing up the web page into sections using CSS, so that a page can be styled to have 2 equal columns, 3 equal columns or columns of varying widths, without using fixed pixel widths, all made easy with Bootstrap or Skeleton. Responsive magic results. You have got to love Skeleton and Bootstrap!

And while I'm thinking about it...

  • White space is important for clarity on the page and should be considered as an important design element.
  • Code -> Test -> Refine ->Code .... is a good pattern for working.
