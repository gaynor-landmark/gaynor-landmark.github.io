Describe what these functions do:

These functions are little pieces of code that do something to, or with the items in a list, better known as an array.

  • .map()

    • 'map' will work its way through each item on the list, in order, transforming that item, using the instructions that have been handed to it, and then using that item to create a new transformed list. For example, the list (array) could contain the names of the children in a family. This might be shown as ["john", "jack", "jill"]. 'map' could be used to take each name in turn and capitalise the first letter or add their surname. The 'transformation' to be done would vary each time. The new list (array) could look like something like this - ["John Smith", "Jack Smith", "Jill Smith" ]. All the items that were in the initial list still appear in the second list.

  • .filter()

    • 'filter' works it way through each item, in order, in the same way that 'map' does. However, instead of transforming each item, the logic that is applied to each is used to decide whether or not that item should appear in the new array. For example, given the list that had just been transformed using 'map', Jack Smith' might be filtered out and the whole new list or array would be filtered down to ["John Smith", "Jill Smith" ].

  • .reduce()

    • 'reduce' is a similar but now flexible tool used for transforming arrays. It also works it's way through each item in the list in order, but it gathers the information on the way. Each time it moves on to a new item, it still has the result of what was created after dealing with the previous item. For example, 'reduce' can be used to find the total of a list of numbers. if this is the original array - [10,20,30], and the total starts with a value of 0, then working from the first item, 0 + 10 = 10. Total is now 10. Now adding the second item, 10 + 20 gives a new Total of 30. On to the last item, 30 + 30 = 60. The Total is 60. This is not the only way 'reduce' can be used. It can be used to create a total or an array or even a new object.

How does recursion compare with iteration:

Both 'recursion' and 'iteration' will repeat a process (block of code) until a condition is met that will stop it repeating, getting closer to the required result each time the repeat occurs. 'Iteration' will generally use a loop in the code within a function, to perform the repetition. 'Recursion' occurs when the whole function performing the process repeats by calling itself (starting itself) again. When using 'iteration' there will be only one call on the stack for the function that the loop is found in, but for 'iteration', a new call is added to the stack each time the function calls itself, without the previous function having completed and being removed from the stack. In both cases errors could occur. Recursion can run out of room on the stack and iteration can cause CPU errors, especially if the exit condition is never reached. Recursion is slower that iteration but is useful when traversing tree structures. Iterative code is generally lenghtier than recursive code.
